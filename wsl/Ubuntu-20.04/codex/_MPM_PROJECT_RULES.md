# MPM 强制协议

## 🚨 死规则 (违反即失败)

1. **改代码前** → 必须先 `code_search` 或 `project_map` 定位，严禁凭记忆改
2. **改代码后** → 必须立即 `memo` 记录
3. **准备改函数时** → 必须先 `code_impact` 分析谁在调用它
4. **code_search 失败** → 必须换词重试（同义词/缩写/驼峰变体），禁止放弃
5. **阅读业务流程时** → 优先使用 `flow_trace`，禁止只看文件名凭感觉推断

---

## 🔧 工具使用时机

- **刚接手陌生项目且无任何代码线索 / 上下文过多需收敛注意力**: 使用 `project_map` / `flow_trace`
- **刚接手项目 / 宏观探索**: 使用 `project_map`
- **理解业务逻辑主链**: 使用 `flow_trace`
- **找具体函数/类的定义**: 使用 `code_search`
- **准备修改某函数**: 使用 `code_impact`
- **代码改完了**: 使用 `memo` (SSOT)

---

## 🚫 禁止

- 禁止凭记忆修改代码
- 禁止 code_search 失败后直接放弃
- 禁止修改代码后不调用 memo
- 禁止并发调用工具
